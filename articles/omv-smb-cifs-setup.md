<h2>Introduction</h2><p>SMB (Server Message Block), also known as CIFS (Common Internet File System), is a widely supported protocol used to share files across different operating systems. This guide will walk you through setting up SMB/CIFS in OpenMediaVault (OMV) so you can access your NAS files from Windows, macOS, or Linux devices.</p><h2>Step 1: Create a Shared Folder</h2><ol><li>Log in to the OMV web interface.</li><li>Navigate to <strong>Storage &gt; File Systems</strong> and make sure your storage volume is mounted.</li><li>Go to <strong>Storage &gt; Shared Folders</strong>.</li><li>Click <strong>Add</strong>, give your shared folder a name (e.g., "Shared"), and select the device and path.</li><li>Click <strong>Save</strong> and then <strong>Apply</strong>.</li></ol><h2>Step 2: Enable SMB/CIFS Service</h2><ol><li>Go to <strong>Services &gt; SMB/CIFS</strong>.</li><li>Check <strong>Enable</strong> and optionally <strong>Enable home directories</strong> if you want per-user folders.</li><li>Click <strong>Save</strong> and then <strong>Apply</strong>.</li></ol><h2>Step 3: Add a Shared Folder to SMB</h2><ol><li>Under <strong>Services &gt; SMB/CIFS &gt; Shares</strong>, click <strong>Add</strong>.</li><li>Select the shared folder you created earlier.</li><li>Give it a name, set access permissions (e.g., Guests Allowed or Authenticated Users), and save.</li><li>Apply the changes when prompted.</li></ol><h2>Step 4: Access from Client Devices</h2><ul><li><strong>Windows:</strong> Press <code>Win + R</code>, type <code>\\OMV-IP-ADDRESS\Shared</code>, and press Enter.</li><li><strong>macOS:</strong> In Finder, press <code>Cmd + K</code>, enter <code>smb://OMV-IP-ADDRESS/Shared</code>, and click Connect.</li><li><strong>Linux:</strong> Open your file manager and enter <code>smb://OMV-IP-ADDRESS/Shared</code> in the address bar.</li></ul><h2>Tips & Tricks</h2><ul><li>Create user accounts under <strong>Access Rights Management</strong> to control access per user.</li><li>Enable logging in the SMB settings for audit and diagnostics.</li><li>Map network drives in Windows for quick access at startup.</li></ul><h2>Troubleshooting</h2><ul><li><strong>Can't connect?</strong> Make sure the OMV firewall isn't blocking SMB (usually ports 137â€“139 and 445).</li><li><strong>macOS won't connect?</strong> Use <code>smb://</code> instead of <code>cifs://</code> and ensure proper permissions.</li><li><strong>Permissions error?</strong> Double-check shared folder ACLs and service-level user permissions.</li></ul><h2>Conclusion</h2><p>Enabling SMB/CIFS in OpenMediaVault allows for seamless file sharing across multiple platforms. Whether you're backing up files or streaming media, SMB gives you the flexibility to make the most of your home NAS. For even more security, consider integrating user authentication and enabling transport encryption if needed.</p>
