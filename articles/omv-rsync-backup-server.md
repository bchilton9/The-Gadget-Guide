<h2>Introduction</h2><p>Rsync is a fast and efficient file-copying tool that can perform incremental backups across local or networked systems. Setting up OpenMediaVault (OMV) as an Rsync server allows you to securely back up files from other computers or servers over your LAN.</p><h2>Step 1: Enable Rsync Service</h2><ol><li>Log into the OMV web interface.</li><li>Navigate to <strong>Services &gt; Rsync &gt; Server</strong>.</li><li>Check <strong>Enable</strong> to turn on the Rsync daemon.</li><li>Click <strong>Save</strong> and then <strong>Apply</strong> the changes.</li></ol><h2>Step 2: Create a Shared Folder</h2><ol><li>Go to <strong>Storage &gt; File Systems</strong> and confirm your disk is mounted.</li><li>Navigate to <strong>Storage &gt; Shared Folders</strong> and click <strong>Add</strong>.</li><li>Name it (e.g., <em>backups</em>), select the device, and path.</li><li>Click <strong>Save</strong> and <strong>Apply</strong>.</li></ol><h2>Step 3: Add an Rsync Module</h2><ol><li>Under <strong>Services &gt; Rsync &gt; Server</strong>, go to the <strong>Modules</strong> tab.</li><li>Click <strong>Add</strong> and configure the module:</li><ul><li><strong>Module name:</strong> backups</li><li><strong>Path:</strong> select your shared folder</li><li><strong>Comment:</strong> Rsync Backup Share</li><li><strong>Auth users:</strong> (optional, for authentication)</li><li><strong>Read only:</strong> No (unless you want read-only access)</li></ul><li>Click <strong>Save</strong> and <strong>Apply</strong>.</li></ol><h2>Step 4: Connect from a Client Device</h2><p>From the device you want to back up, run:</p><pre><code>rsync -avz /path/to/files/ user@omv-ip::backups</code></pre><p>Use <code>rsync://omv-ip/backups</code> in backup tools that support Rsync over daemon protocol.</p><h2>Tips & Tricks</h2><ul><li>Use <code>--delete</code> with caution to mirror deletions from the source.</li><li>Combine with cron jobs or backup software for scheduled automation.</li><li>Enable logging to monitor activity: <strong>Services &gt; Rsync &gt; Server</strong> settings.</li></ul><h2>Troubleshooting</h2><ul><li><strong>Connection refused?</strong> Make sure the Rsync service is enabled and OMV firewall isnâ€™t blocking the port (default: 873).</li><li><strong>Permission denied?</strong> Double-check Rsync module settings and user credentials if auth is used.</li><li><strong>Backups not updating?</strong> Try adding <code>--checksum</code> or <code>--ignore-times</code> to the Rsync command.</li></ul><h2>Conclusion</h2><p>Using Rsync with OpenMediaVault provides a lightweight and highly customizable backup solution. With its support for incremental updates, automation, and remote access, it's a solid option for both personal and professional NAS setups.</p>
