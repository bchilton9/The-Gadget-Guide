<h2>Introduction</h2><p>Uptime Kuma is a free and open-source monitoring tool that runs in your local environment. With a user-friendly interface and great functionality, it's the perfect choice for self-hosted uptime monitoring. This guide walks you through deploying it in Docker on your OpenMediaVault system.</p><h2>Step 1: Prepare Your System</h2><ol><li>Ensure Docker and Portainer are already installed on OMV. If not, install them using OMV-Extras.</li><li>Log into Portainer (usually at http://omv.local:9000).</li></ol><h2>Step 2: Create a New Docker Container</h2><ol><li>In Portainer, go to <strong>Containers > Add container</strong>.</li><li>Set the name to <code>uptime-kuma</code>.</li><li>Image: <code>louislam/uptime-kuma</code>.</li><li>Under <strong>Network</strong>, use <code>bridge</code> or create a custom network if desired.</li><li>Under <strong>Ports</strong>:<ul><li>Host: <code>3001</code></li><li>Container: <code>3001</code></li></ul></li><li>Under <strong>Volumes</strong>:<ul><li>Map a volume from host: <code>/srv/dev-disk-by-label-data/appdata/uptime-kuma</code></li><li>To container: <code>/app/data</code></li></ul></li><li>Click <strong>Deploy the container</strong>.</li></ol><h2>Step 3: Access Uptime Kuma</h2><p>After a few seconds, go to <code>http://omv.local:3001</code> in your browser. You’ll be greeted with the setup wizard to create your admin account and begin adding services.</p><h2>Tips & Tricks</h2><ul><li>Enable notifications via Discord, Telegram, Email, etc. under the <strong>Settings > Notifications</strong> panel.</li><li>Use HTTP(s), TCP, DNS, and Ping checks depending on your service type.</li><li>Create multiple status pages for different audiences.</li></ul><h2>Troubleshooting</h2><ul><li><strong>Can’t access port 3001?</strong> Make sure the container is running and no other service is using that port.</li><li><strong>Data not persisting?</strong> Double-check your volume path on the host side.</li><li><strong>Container won’t start?</strong> Use <code>docker logs uptime-kuma</code> for error messages or review Portainer logs.</li></ul><h2>Conclusion</h2><p>Uptime Kuma brings professional-grade service monitoring to your home lab with an easy setup via Docker and OMV. It’s highly configurable, visually appealing, and light on resources. If you run self-hosted apps, it’s a must-have tool for peace of mind.</p>
