<h2>Introduction</h2><p>File Transfer Protocol (FTP) allows users to upload and download files from your OMV NAS. While FTP is a widely supported protocol, it requires proper setup and security precautions—especially if you intend to access your NAS externally.</p><h2>Step 1: Enable FTP Service</h2><ol><li>Log in to the OMV web interface.</li><li>Navigate to <strong>Services &gt; FTP</strong>.</li><li>Check <strong>Enable</strong> and set your desired port (default: 21).</li><li>Optionally enable <strong>Passive Mode</strong> and define a port range (e.g., 50000–51000).</li><li>Click <strong>Save</strong> and <strong>Apply</strong>.</li></ol><h2>Step 2: Configure Shared Folder Permissions</h2><ol><li>Go to <strong>Storage &gt; Shared Folders</strong> and select the folder you want to expose via FTP.</li><li>Ensure the appropriate user or group has <strong>read/write</strong> access.</li></ol><h2>Step 3: Create or Assign FTP User</h2><ol><li>Navigate to <strong>Access Rights Management &gt; Users</strong>.</li><li>Add a new user or modify an existing one, ensuring it belongs to the <code>ftp</code> group or has home directory access to the shared folder.</li></ol><h2>Step 4: Test FTP Access Locally</h2><p>Use an FTP client like FileZilla and connect using:</p><ul><li><strong>Host:</strong> Your OMV IP address</li><li><strong>Port:</strong> 21 (or your custom port)</li><li><strong>User/Pass:</strong> OMV credentials</li></ul><h2>Step 5: Secure External Access (Optional)</h2><ol><li>Set up port forwarding on your router for port 21 (and passive range if configured).</li><li>Use a Dynamic DNS (DDNS) service if you don't have a static IP.</li><li>Consider using <strong>FTP over TLS</strong> for encrypted connections (enabled in OMV settings).</li></ol><h2>Tips & Tricks</h2><ul><li>Enable chroot jail to restrict users to their home directories.</li><li>Set upload/download bandwidth limits to prevent saturation of your internet connection.</li><li>Use a firewall to restrict access to known IP addresses if FTP is used externally.</li></ul><h2>Troubleshooting</h2><ul><li><strong>Can’t connect?</strong> Ensure the FTP service is enabled and that the port isn’t blocked by your firewall or ISP.</li><li><strong>File permissions errors?</strong> Double-check shared folder permissions and the user group assignments.</li><li><strong>Timeouts?</strong> Use passive mode and forward passive port ranges on your router.</li></ul><h2>Conclusion</h2><p>FTP is a flexible method for remote file access on your OMV NAS. When set up correctly—with security features like TLS and restricted access—it becomes a valuable tool for backups, media management, and remote uploads.</p>
